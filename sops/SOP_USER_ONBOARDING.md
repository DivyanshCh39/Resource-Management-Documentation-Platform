## 1. Purpose

This SOP establishes the standard process for onboarding new users to the Resource Management Platform, ensuring consistent, efficient, and secure access provisioning.

---

## 2. Scope

This procedure applies to:
- All new platform users
- Internal employees
- External collaborators
- Temporary access requests
- Contractor access provisioning

---

## 3. Responsibilities

| Role | Responsibility |
|------|----------------|
| **Support Team** | Process onboarding requests, create accounts |
| **IT Security** | Approve access levels, verify security compliance |
| **Department Manager** | Authorize user access requests |
| **New User** | Complete required training, acknowledge policies |

---

## 4. Prerequisites

Before initiating onboarding:
- ✓ User access request form submitted
- ✓ Manager approval received
- ✓ Security clearance verified (if applicable)
- ✓ Department/team assignment confirmed
- ✓ Access level determined

---

## 5. Procedure

### Step 1: Access Request Submission

**Timeline**: Day 0  
**Responsible**: Department Manager

1. Manager completes User Access Request Form
2. Form includes:
   - User full name
   - Email address
   - Department/Team
   - Required access level
   - Business justification
   - Expected duration of access
3. Submit form to support team via ticketing system
4. Ticket priority: Medium
5. Expected response: Within 4 hours

**Forms Required**:
- User Access Request Form
- Security Acknowledgment (if applicable)

---

### Step 2: Request Review and Approval

**Timeline**: Day 0-1  
**Responsible**: Support Team + IT Security

1. Support team reviews submitted request
2. Verify all required fields completed
3. Confirm manager authorization
4. Check for duplicate accounts
5. Validate email address format
6. Review access level appropriateness
7. Forward to IT Security for approval (if required)
8. IT Security reviews and approves within 2 hours
9. Support team receives approval confirmation

**Approval Criteria**:
- Valid business justification
- Appropriate access level
- Manager authorization present
- No security flags
- Email domain approved

---

### Step 3: Account Creation

**Timeline**: Day 1  
**Responsible**: Support Team

1. Create user account in system
2. Generate temporary credentials:
   - Username: firstname.lastname
   - Temporary password: Auto-generated secure password
3. Assign appropriate access level:
   - **Basic User**: File upload/download, ticket creation
   - **Power User**: + File deletion, advanced features
   - **Admin**: Full system access
4. Set account status: Pending Activation
5. Configure user permissions based on access level
6. Verify account creation successful

**Quality Check**:
- Username follows naming convention
- Password meets security requirements (12+ chars, mixed case, special chars)
- Permissions correctly assigned
- Account visible in user management system

---

### Step 4: Welcome Email and Credentials

**Timeline**: Day 1  
**Responsible**: Support Team

1. Send welcome email to new user containing:
   - Platform URL
   - Temporary username
   - Temporary password (sent separately for security)
   - Password change requirement notice
   - Login instructions
   - User Guide link
   - Support contact information
2. CC manager on welcome email
3. Log credential distribution in system
4. Mark ticket status: Credentials Sent

**Email Template**:
```
Subject: Welcome to Resource Management Platform - Login Credentials

Dear [User Name],

Welcome to the Resource Management Platform! Your account has been created.

Platform URL: [URL]
Username: [username]

Your temporary password will be sent in a separate email for security purposes.

You will be required to change your password upon first login.

Resources:
- User Guide: [Link]
- Support: Submit ticket through platform

For assistance, please contact our support team.

Best regards,
Platform Support Team
```

---

### Step 5: Initial Login and Setup

**Timeline**: Day 1-2  
**Responsible**: New User

1. User receives credentials
2. User navigates to platform URL
3. Enter temporary credentials
4. System prompts for password change
5. User creates new secure password
6. Password must meet requirements:
   - Minimum 12 characters
   - At least one uppercase letter
   - At least one lowercase letter
   - At least one number
   - At least one special character
7. User agrees to Terms of Service
8. User completes first-time setup wizard
9. Account status changes to: Active

**User Actions Required**:
- Change temporary password
- Acknowledge Terms of Service
- Complete profile information
- Review User Guide

---

### Step 6: Training and Documentation

**Timeline**: Day 2-3  
**Responsible**: New User

1. User reviews comprehensive User Guide
2. Watch introductory video tutorials (if available)
3. Review relevant SOPs for their role
4. Familiarize with platform features:
   - File upload/download
   - Ticket creation
   - Documentation access
5. Practice with test files (optional)
6. Complete knowledge check quiz
7. Certificate of completion generated

**Training Materials**:
- User Guide (comprehensive)
- Video tutorials (feature-specific)
- SOPs library
- FAQ document
- Quick reference card

---

### Step 7: Verification and Confirmation

**Timeline**: Day 3  
**Responsible**: Support Team

1. Verify user first login successful
2. Confirm password changed from temporary
3. Check Terms of Service acceptance
4. Verify training completion (if mandatory)
5. Send follow-up email to user
6. Contact manager to confirm successful onboarding
7. Close onboarding ticket
8. File documentation in user records

**Follow-up Email Template**:
```
Subject: Platform Onboarding Complete

Dear [User Name],

Your onboarding to the Resource Management Platform is complete.

✓ Account activated
✓ Password set
✓ Training accessed

If you have any questions or need assistance, please submit a ticket through the platform.

Thank you,
Platform Support Team
```

## 9. Troubleshooting

### Common Issues and Resolutions

**Issue**: User cannot receive welcome email  
**Resolution**: Check spam folder, verify email address, resend manually

**Issue**: Temporary password doesn't work  
**Resolution**: Reset password, generate new temporary credentials

**Issue**: User cannot change password  
**Resolution**: Verify password meets requirements, check for special character issues

**Issue**: Account shows as pending after 24 hours  
**Resolution**: Verify all approvals received, manually activate if needed

**Issue**: User cannot access certain features  
**Resolution**: Verify access level assignment, adjust permissions if needed
